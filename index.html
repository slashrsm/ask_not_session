“We are what we repeatedly do. Excellence, therefore, is not an act, but a habit.”

Aristotle

21 days, 6 weeks, 90 days whatever. Point is that you need to repeat something for some time in order to make it a habbit.

Treat as if it would be own project

- Experience a bug: - Check if it is reported - Report it if not - If it is high prio for you fix it and submit a patch - If you don’t do that be prepared for it to never be fixed - if it is low prio for you then why should someone else fix it? - This process needs to become a habbit for your developers - if this is not the case you are not part of the community

Community tools are your tools

- Issue queues - IRC - Missing functionality in core/existing module - Don’t complain!!! - Feature request - Low prio: Maybe someone else does it - but be prepared that it probably won’t - High prio: Write a patch/module - Talk about it - maybe you find someone that needs it too and you share the load, collaborate - Don’t do custom modules (unless it is very custom glue code)

Don't wait "Until it is ready"

- "Is media ready?" - It won't happen automagically! - Requires real people to do real work! - Missing functionality in core/existing module - Don’t complain!!! - Feature request - Low prio: Maybe someone else does it - but be prepared that it probably won’t - High prio: Write a patch/module - Talk about it - maybe you find someone that needs it too and you share the load, collaborate - Don’t do custom modules (unless it is very custom glue code)

Be selfish

- Make your solutions general so you (or someone else) can use it in the future - Share them under free sw license - Since other people will be able to use it you might get new functionality for free.

Bigger initiatives...

- are a bit different - More complex, consumes much more time - You always need to collaborate and communicate - At this point it is not just about development (blog posts, conference talks, management, motivation) - Otherwise you likely end up with project of type 5 - Start with steps above, climb the ladder and when you gain experience do this - We still don't good process for this IMO - enthusiastic individuals, burnout, not sustainable
